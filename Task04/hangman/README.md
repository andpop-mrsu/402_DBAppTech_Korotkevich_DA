# Игра "Виселица" (Hangman) - Версия с базой данных SQLite

Консольная реализация классической игры "Виселица" на PHP с полной поддержкой SQLite базы данных.

## Требования
- PHP 7.4 или выше
- Расширения: PDO, PDO_SQLite
- Composer для управления зависимостями

## Установка

### Установка из репозитория:
```bash
git clone https://github.com/DmitriyKorotkevich/402_DBAppTech_Korotkevich_DA.git
cd 402_DBAppTech_Korotkevich_DA/Task04/hangman
composer install
chmod +x bin/hangman
Установка через Packagist (глобально):
bash
composer global require dmitriykorotkevich/hangman
После глобальной установки добавьте в PATH:

bash
# Windows (Git Bash):
export PATH="$PATH:/c/Users/$USER/AppData/Roaming/Composer/vendor/bin"
Использование
Параметры командной строки:
text
hangman [ПАРАМЕТРЫ]

Параметры:
  -n, --new          Новая игра (режим по умолчанию)
  -l, --list         Показать список всех сохраненных игр
  -r, --replay ID    Повторить игру с указанным ID
  -h, --help         Показать справку
Примеры:
bash
# Новая игра (запросит имя игрока)
hangman

# Показать список всех сохраненных игр
hangman --list
hangman -l

# Повторить игру #5
hangman --replay 5
hangman -r 5

# Справка
hangman --help
hangman -h
Игровой процесс
Новая игра:
Программа запрашивает имя игрока

Компьютер выбирает случайное слово из 6 букв из базы данных

Игрок вводит буквы по одной

За каждую ошибку рисуется часть виселицы

У игрока есть 6 попыток

Результат игры автоматически сохраняется в базу данных

Формат ввода:
Вводите по одной букве за ход

Можно ввести всё слово целиком

Регистр не имеет значения (A = a)

Для выхода введите: exit или quit

Псевдографика:
Игра использует ASCII-графику для отображения виселицы:

text
+---+    +---+    +---+    +---+    +---+    +---+    +---+
    |        0   |    0   |    0   |    0   |    0   |    0   |
    |            |    |   |   /|   |   /|\  |   /|\  |   /|\  |
    |            |        |        |        |   /    |   / \  |
   ===          ===      ===      ===      ===      ===      ===
   0 ошибок     1 ошибка 2 ошибки 3 ошибки 4 ошибки 5 ошибок 6 ошибок
Работа с базой данных
Структура базы данных:
Игра использует SQLite базу данных (файл: data/hangman.db).

Таблицы:
games - информация об играх:

id - уникальный идентификатор

player_name - имя игрока

secret_word - загаданное слово

is_won - результат (1 - победа, 0 - поражение)

played_at - дата и время игры

attempts - попытки в играх:

id - уникальный идентификатор

game_id - ссылка на игру

attempt_number - номер попытки

letter - введенная буква или слово

is_correct - результат (1 - правильно, 0 - неверно)

words - словарь слов:

id - уникальный идентификатор

word - слово (6 букв)

Автоматическое сохранение:
При завершении игры (победа или поражение) автоматически сохраняется:

Информация об игре (игрок, слово, результат, дата)

Все попытки с указанием правильности

Просмотр истории:
Команда hangman --list показывает список всех сохраненных игр в формате:

text
Игра #1   | Игрок: Dmitriy      | Слово: PYTHON  | ПОБЕДА     | Попыток: 4   | 22.12.2023 15:30
Игра #2   | Игрок: Anna         | Слово: KOTLIN  | ПОРАЖЕНИЕ  | Попыток: 6   | 22.12.2023 16:45
Повтор игры:
Команда hangman --replay ID воспроизводит игру по шагам:

Показывает информацию об игре

Поочередно отображает все попытки

Показывает состояние слова после каждой попытки

Отображает виселицу по мере ошибок

Архитектура проекта
Проект разработан с использованием паттерна MVC (Model-View-Controller):

Модели (Models/):
Database.php - подключение и работа с SQLite

Game.php - основная игровая логика

GameRecord.php - работа с сохраненными играми

Представление (View/):
View.php - отображение игры, псевдографика, вывод информации

Контроллер:
Controller.php - управление игровым процессом и режимами

Структура базы данных:
text
